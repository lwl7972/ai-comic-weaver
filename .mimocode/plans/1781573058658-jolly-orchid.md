# Plan: 后端启动完成后再加载前端页面

## Problem

Electron 打包应用启动时，前端 Vue 页面在后端 Spring Boot 还没完全启动（约 10 秒）时就加载了，导致前端立即发起 API 请求全部失败（`ERR_CONNECTION_REFUSED`）。用户看到大量连接错误后才慢慢恢复正常。

## Current State (electron/main.js)

```
app.whenReady() → detectFreePort() → createWindow() → startBackend() → setupAutoUpdater()
```

- `createWindow()` 第 96 行调用 `showLoadingScreen(mainWindow)` 显示加载页
- `startBackend()` spawn Java 进程，监听 stdout 中的 `Started AiComicPlatformApplication`
- 检测到后端就绪后：`mainWindow.webContents.send('backend-ready', ...)` + `loadApp(mainWindow)`（第 252-253 行）
- `loadApp()` 加载 Vue 前端 → 前端 mount 后立即调用 API → 后端可能还没就绪

**问题根因**：`loadApp()` 在 stdout 检测到字符串时触发，但 Spring Boot 的 `Started AiComicPlatformApplication` 日志输出并不意味着 HTTP 端口已经可以接受连接。加上前端加载、JS 解析、mount 后立即调 API，存在竞态。

## Solution

### 核心改动：`electron/main.js`

**1. 后端就绪检测改为 HTTP 健康检查（替代 stdout 字符串匹配）**

在 `startBackend()` 中，当 stdout 检测到 `Started AiComicPlatformApplication` 后，不立即调用 `loadApp()`，而是用 `http.get` 请求 `http://localhost:${backendPort}/api/v1/app-config`，确认端口真正可接受连接后再加载前端。

```js
// startBackend() 中 stdout 检测到后：
if (!backendReadyFired && output.includes('Started AiComicPlatformApplication')) {
  backendReadyFired = true
  // 不再立即 loadApp，而是启动健康检查轮询
  waitForBackendReady(backendPort, mainWindow)
}

function waitForBackendReady(port, win, retries = 0) {
  const http = require('http')
  const req = http.get(`http://127.0.0.1:${port}/api/v1/app-config`, { timeout: 2000 }, (res) => {
    if (res.statusCode === 200 || res.statusCode === 401) {
      backendReady = true
      clearStartupTimeout()
      console.log('[Electron] Backend HTTP ready')
      win.webContents.send('backend-ready', { port })
      loadApp(win)
    } else {
      retryOrFail()
    }
  })
  req.on('error', () => retryOrFail())
  req.on('timeout', () => { req.destroy(); retryOrFail() })

  function retryOrFail() {
    if (retries < 10) {
      setTimeout(() => waitForBackendReady(port, win, retries + 1), 500)
    } else {
      console.error('[Electron] Backend HTTP check failed after retries')
      win.webContents.send('backend-error', { message: '后端启动超时', canRestart: true })
    }
  }
}
```

**2. 修复 loading screen 的 IPC listener**

`showLoadingScreen()` 中的 `require('electron')` 在 `contextIsolation: true` 下不工作。改为通过 preload 注入的 `window.electronAPI` 监听事件：

```js
// loading screen 内联脚本改为：
<script>
  if (window.electronAPI) {
    window.electronAPI.onBackendError((data) => {
      const el = document.getElementById('error');
      if (el) { el.style.display='block'; el.textContent = data.message || '后端启动失败'; }
    });
  }
</script>
```

**3. 前端增加防御性等待（可选但推荐）**

在 `frontend/src/utils/http.ts` 中，添加后端就绪检查。当 `electronAPI` 可用时，等待 `backend-ready` 信号后再发起请求：

```ts
// http.ts 顶部添加
let backendReadyPromise: Promise<void> | null = null

if ((window as any).electronAPI?.onBackendReady) {
  backendReadyPromise = new Promise<void>((resolve) => {
    (window as any).electronAPI.onBackendReady(() => resolve())
  })
}

// http interceptor 中添加等待
http.interceptors.request.use(async (config) => {
  if (backendReadyPromise) await backendReadyPromise
  // ... existing logic
})
```

### Files to Modify

1. **`electron/main.js`** — 核心改动：HTTP 健康检查替代 stdout 匹配 + 修复 loading screen IPC
2. **`frontend/src/utils/http.ts`** — 添加后端就绪等待（防御性，确保前端不在后端未就绪时发请求）
